# scripts/ — Data pipeline

Produces [`pwa/src/data/events.json`](../pwa/src/data/events.json) — the single source
of truth the PWA consumes at build time.

Everything here runs **anonymously against public endpoints**. No Marvel/Disney
authentication is used during scraping or enrichment.

## Pipeline at a glance

```text
  ┌──────────────┐   ┌─────────────────┐   ┌──────────────┐   ┌──────────┐
  │ scrape-cbh   │──►│ scrape-wikipedia│──►│ merge-manual │──►│ gen-runs │
  └──────────────┘   └─────────────────┘   └──────────────┘   └────┬─────┘
                                                                    ▼
                     ┌──────────────────────────────────────────────┐
                     │              events.json (raw)               │
                     └─────────────────┬────────────────────────────┘
                                       ▼
                     ┌──────────────────────────────────────────────┐
                     │  enrich-from-sitemap                         │
                     │  · Marvel.com sitemap → slug, marvelId       │
                     │  · marvel.com/comics/issue/{id}/{slug}       │
                     │    → digitalId (for native deeplink)         │
                     │  · bifrost.marvel.com → DRN + cover URL      │
                     └─────────────────┬────────────────────────────┘
                                       ▼
                     ┌──────────────────────────────────────────────┐
                     │  enrich-from-comicvine (optional fallback)   │
                     │  Fills cover URLs for series Marvel doesn't  │
                     │  index on their public sitemap.              │
                     └─────────────────┬────────────────────────────┘
                                       ▼
                         events.json (ready to commit)
```

## Run it

```bash
cd scripts
npm install

# Full scrape+merge+enrich (writes to ../pwa/src/data/events.json in place):
npm run pipeline

# Optional fallback for covers the native enricher missed:
CV_API_KEY=xxx npm run enrich:comicvine
```

## Source scripts

| Script | Purpose |
|--------|---------|
| `scrape-cbh.ts` | Comic Book Herald per-page scraper. Configured per event/character/team at the top of the file. |
| `scrape-wikipedia.ts` | Wikipedia scraper for events CBH doesn't cover cleanly (Age of X, AXIS, BotA, Schism, Second Coming, Utopia, Dark X-Men, Fear Itself, Curse of the Mutants, Necrosha, From the Ashes, Age of Revelation). Merges non-destructively. |
| `merge-manual.ts` | Reads `manual-guides.json` and merges hand-curated guides (Cyclops, West Coast Avengers) into events.json. |
| `gen-runs.ts` | Generates creative-team **run** events (Morrison's New X-Men, Whedon's Astonishing, Brubaker/Fraction/Gillen Uncanny, Aaron's Wolverine and the X-Men, Bendis' All-New + Uncanny). Each run has its own Event entry with category `run`. |

## Enrichment scripts

| Script | Enriches | Source | Rate-limited |
|--------|----------|--------|--------------|
| `enrich-from-sitemap.ts` | `marvelId`, `slug` (from `marvel.com/sitemap-comics-{0,1}.xml`), `digitalId` (from each issue page), `drn` + `cover` (from `bifrost.marvel.com/unison/legacy?digitalId=N`) | marvel.com + bifrost | Yes — marvel.com rate-limits by IP. Concurrency=2, 1.5–3s delay, autosave every 25 hits, User-Agent rotation on 403. Use `--only-sitemap` for a fast pass that only fills slug/marvelId. |
| `enrich-from-comicvine.ts` | `cover` (fallback for issues the native enricher missed) | comicvine.gamespot.com | 200 req/hour per endpoint. Batches by series volume to minimise calls. Requires `CV_API_KEY` env. |

## Data control files

| File | What it contains |
|------|------------------|
| `manual-guides.json` | Hand-curated reading guides (Cyclops, WCA). Each guide is a full event-shaped entity with issues, notes, category. |
| `overrides.json` | Per-issue metadata overrides applied by `enrich-from-sitemap.ts`. Use this for issues where auto-matching fails — e.g. "Sentinel Squad" → slug `sentinel_squad_one_2006_N`. |
| `.enrichment-cache.json` | Auto-generated by `lib-enrichment-cache.ts`. Preserves slug/digitalId/DRN/cover across scraper re-runs so re-scraping CBH doesn't wipe hours of enrichment work. Gitignored. |

## Preserving enrichment across re-scrapes

`scrape-cbh.ts` writes `events.json` from scratch — this wipes any prior enrichment.
The cache prevents that loss:

```bash
npm run cache:snapshot   # snapshot current enrichment into .enrichment-cache.json
npm run scrape:cbh       # rewrites events.json with fresh issue data
npm run cache:restore    # re-applies snapshotted marvelId/slug/digitalId/DRN/cover by (title, number, year)
npm run enrich:sitemap   # fills only the new gaps
```

## Public sources used

| Source | Endpoint | Access |
|--------|----------|--------|
| Comic Book Herald | `comicbookherald.com/{slug}` | Anonymous HTTP |
| Wikipedia | `en.wikipedia.org/wiki/{article}` | Anonymous HTTP |
| Marvel sitemap | `www.marvel.com/sitemap-comics-{0,1}.xml` | Anonymous HTTP |
| Marvel issue catalog | `www.marvel.com/comics/issue/{id}/{slug}` | Anonymous HTTP |
| Marvel bifrost (GraphQL) | `bifrost.marvel.com/unison/legacy?digitalId=N` | Anonymous, unauthenticated |
| ComicVine API | `comicvine.gamespot.com/api/*` | Free API key, 200 req/h |

## Coverage

Current metadata coverage (as of last run):

- **Slug** (marvelId + canonical URL): ~81%
- **DigitalId** (native MU deeplink): ~78%
- **DRN** (`marvelunlimited://issue/{drn}` native scheme): ~71%
- **Cover** (thumbnail image): ~81%

Gaps come from: series not indexed on marvel.com's public sitemap (Wolverine and the X-Men,
Infinity Comics), future/unreleased 2025–2026 issues, Marvel Voices one-shots, and event-specific
one-shots with non-canonical titles. See `overrides.json` for hand-fixed entries.
